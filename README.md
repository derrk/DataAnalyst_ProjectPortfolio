# Operations Automation \& Data Center Analytics Portfolio



This repository contains a portfolio of automation, analytics, and reporting projects built to support large-scale data center and infrastructure operations. The primary goal of this work is to reduce manual effort, improve data consistency, and provide clear operational insights through repeatable, validated workflows.



While many of these projects originated from Bitcoin mining operations, the tooling and methodologies are intentionally designed to be infrastructure-agnostic and directly applicable to high-performance computing (HPC) and data center environments.



---



## Core Objectives



- Automate repetitive operational reporting and analysis

- Improve visibility into asset health, availability, and lifecycle status

- Enable data-driven decision-making around capacity, consolidation, and risk

- Provide reproducible, auditable analytics suitable for production environments



---



## Portfolio Overview



Each project within this repository is self-contained and includes:

- A clear problem statement

- Defined inputs and outputs

- Validation steps to ensure data integrity

- Example output using sanitized or sample data



### Projects



| ID | Project | Description |

|----|--------|-------------|

| 01 | Daily Performance Reporting | Automated generation of daily site-level performance and availability summaries |

| 02 | Warranty Expiration Tracker | Identification and prioritization of assets approaching or exceeding warranty coverage |

| 03 | Fleet Health Summary | Aggregated hardware health metrics, including failure counts and model-level rollups |

| 04 | Asset Inventory Sync | CMDB-style reconciliation logic modeling asset state transitions and source-of-truth alignment |

| 05 | Consolidation Planner | Capacity and slot recovery analysis to support consolidation and modernization efforts |
